אנו ממשיכים בדרך שלנו אל ההוכחה שלא קיים אלגוריתם שפותר את הבעיה העשירית של הילברט. אני מניח שקראתם את <a href="http://www.gadial.net/2012/08/27/hilbert_tenth_intro/">פוסט המבוא</a> ולכן קופץ ישר לעניינים הפעם. המושג המרכזי בפוסט הקודם היה <strong>קבוצה דיופנטית</strong>. זוהי קבוצה {% equation %}S=\left\{ \left(a_{1},\dots,a_{n}\right)\right\} {% endequation %} של {% equation %}n{% endequation %}-יות של מספרים טבעיים חיוביים, כך שקיים פולינום {% equation %}p\left(x_{1},\dots,x_{n},y_{1},\dots,y_{m}\right){% endequation %} בעל התכונה הבאה: למשוואה הדיופנטית {% equation %}q\left(y_{1},\dots,y_{m}\right)=0{% endequation %}, עבור {% equation %}q\left(y_{1},\dots,y_{m}\right)=p\left(a_{1},\dots,a_{n},y_{1},\dots,y_{m}\right){% endequation %} יש פתרון בשלמים חיוביים אם ורק אם {% equation %}\left(a_{1},\dots,a_{n}\right)\in S{% endequation %}. אפשר גם לעשות ההפך - להתחיל מפולינום {% equation %}p\left(x_{1},\dots,x_{n},y_{1},\dots,y_{m}\right){% endequation %} ולסמן קבוצה {% equation %}S_{p}{% endequation %} שכוללת את כל ה-{% equation %}n{% endequation %}-יות שכאשר מציבים אותם במקום ה-{% equation %}x{% endequation %}-ים, מקבלים משוואה דיופנטית עם פתרון בשלמים חיוביים.

אם לא הבנתם כלום ממה שכתבתי כרגע, נסו לקרוא שוב את המבוא.

בפוסט הקודם ראינו שאפשר להגדיר יחסים מוכרים באופן הזה, למשל את יחס החלוקה (שהוא קבוצת הזוגות {% equation %}\left(a,b\right){% endequation %} כך ש-{% equation %}a|b{% endequation %}, כלומר {% equation %}a{% endequation %} מחלק את {% equation %}b{% endequation %}, כלומר קיים {% equation %}c{% endequation %} שלם כך ש-{% equation %}b=ac{% endequation %}). בואו נראה לצורך החימום עוד דוגמה - היחס "קטן-שווה", הידוע גם כ-{% equation %}a\le b{% endequation %}. נגדיר אותו באופן הבא:

{% equation %}x\le y\iff\exists z\left(x+z-1=y\right){% endequation %}

רגע, מה? מה הולך כאן? מה אלו הסימונים הללו? אני עובר לשיטה שבה מרטין דיוויס משתמש במאמר שלו כי היא אלגנטית וקצרה יותר. כאן הפולינום שלי הוא {% equation %}p\left(x,y,z\right)=x+z-1-y{% endequation %}, והמשתנים שבהם צריך להציב ערכים הם {% equation %}x,y{% endequation %} ואז לראות אם למשוואה הדיופנטית שנותרה לנו, שיש בה רק את המשתנה {% equation %}z{% endequation %}, יש פתרון. די ברור בסימון של דיוויס מי המשתנים ששייכים לקבוצה הראשונה (אלו שבאגף שמאל של החצים) ומי המשתנים שבקבוצה השניה (אלו שיש סימן {% equation %}\exists{% endequation %} - "קיים" לפניהם) והיכן הפולינום עצמו (מה שבסוגריים, עד כדי העברת אגפים אם יש צורך). אז אני מקווה שאנחנו מיודדים עם הסימון הזה כעת וגם מבינים למה הפולינום שלעיל אכן מגדיר את {% equation %}x\le y{% endequation %}.

עכשיו, תרגיל - מה השתנה כאן?

{% equation %}x&lt;y\iff\exists z\left(x+z=y\right){% endequation %}

ולמה?

עכשיו בואו נעבור למשהו קצת יותר מתוחכם. ראינו כבר שיש פולינום {% equation %}p_{1}{% endequation %} שמגדיר את הקבוצה {% equation %}S_{p_{1}}=\left\{ \left(a,b\right)\ |\ a|b\right\} {% endequation %}. ראינו גם שיש פולינום {% equation %}p_{2}{% endequation %} שמגדיר את הקבוצה {% equation %}S_{p_{2}}=\left\{ \left(a,b\right)\ |\ a\le b\right\} {% endequation %}. האם יש פולינום שמגדיר את הקבוצה {% equation %}S=\left\{ \left(a,b\right)\ |\ a\le b\wedge a|b\right\} {% endequation %} ({% equation %}\wedge{% endequation %} הוא הסימן של "וגם"; ו-{% equation %}\vee{% endequation %} הוא הסימן של "או"), כלומר הקבוצה {% equation %}S_{p_{1}}\cap S_{p_{2}}{% endequation %}? התשובה חיובית: הפולינום {% equation %}p_{1}^{2}+p_{2}^{2}{% endequation %}. למה? כי אחרי הצבת ערכים בו, נשאר עם משוואה דיופנטית שהדרך היחידה שלה להתאפס היא שגם {% equation %}p_{1}^{2}{% endequation %} יתאפס וגם {% equation %}p_{2}^{2}{% endequation %} יתאפס (כי אם אחד מהם לא מתאפס, אז אחרי העלאה בריבוע הוא חיובי ולכן הסכום יהיה גדול מאפס). את העניין הזה אפשר כמובן להכליל לחיתוך על מספר סופי כלשהו של נחתכים. המסקנה: מחלקת הקבוצות הדיופנטיות סגורה לחיתוכים סופיים. ומה עם איחודים סופיים? גם אותם אפשר לקבל. הקבוצה {% equation %}S_{p_{1}}\cup S_{p_{2}}{% endequation %} מוגדרת על ידי הפולינום {% equation %}p_{1}p_{2}{% endequation %} - הוכיחו זאת לעצמכם.

אם אנחנו כבר עוסקים במשפטים "כלליים" על קבוצות דיופנטיות, הנה משפט מעניין במיוחד: קבוצה {% equation %}S{% endequation %} של טבעיים חיוביים היא דיופנטית אם ורק אם קיים פולינום {% equation %}p{% endequation %} כך ש-{% equation %}S{% endequation %} היא בדיוק קבוצת הערכים <strong>החיוביים</strong> ש-{% equation %}p{% endequation %} מחזיר (לכל קלט שהוא).

זה משפט מעניין למדי כי הוא מוציא מהתמונה לגמרי את עניין ה"בדוק אם משוואה דיופנטית כלשהי היא פתירה". כמובן שזה לא עושה את החיים קלים במיוחד - קודם, כדי לברר אם איבר כלשהו שייך לקבוצה, היה צריך להציב אותו בפולינום שהגדיר את הקבוצה ולקוות שאפשר יהיה להבין אם המשוואה הדיופנטית שנקבל היא פתירה או לא; עכשיו בכלל לא ברור מה אפשר לעשות חוץ מלהציב ערכים ב-{% equation %}p{% endequation %} ולהתפלל שהאיבר שלנו יצוץ מתישהו או שנבין למה הוא לא יכול לצוץ אף פעם.

איך מוכיחים את המשפט? זה תרגיל נחמד. קחו רגע לחשוב עליו לפני שאתם קוראים את ההוכחה.

נתחיל מהכיוון הקל. נניח שיש לנו קבוצה {% equation %}S{% endequation %} שמוגדרת על ידי {% equation %}p\left(y_{1},\dots,y_{n}\right){% endequation %} בדרך ה"חדשה" שתיארתי - {% equation %}S{% endequation %} היא קבוצת הערכים החיוביים ש-{% equation %}p{% endequation %} יכול להחזיר. אנחנו רוצים למצוא פולינום {% equation %}q{% endequation %} כך ש-{% equation %}S=S_{q}{% endequation %}, כאשר {% equation %}S_{q}{% endequation %} מוגדרת בשיטה הישנה (ערכים שאחרי שמציבים אותם ב-{% equation %}q{% endequation %} מקבלים משוואה דיופנטית פתירה). מה נגדיר? פשוט מאוד: {% equation %}q\left(x,y_{1},\dots,y_{n}\right)=x-p\left(y_{1},\dots,y_{n}\right){% endequation %}. ברור שכאשר {% equation %}a\in S{% endequation %} אז {% equation %}q\left(a,y_{1},\dots,y_{m}\right){% endequation %}, שמגדיר את המשוואה {% equation %}a=p\left(y_{1},\dots,y_{n}\right){% endequation %} יהיה פתיר; וכמובן שאם {% equation %}a=p\left(y_{1},\dots,y_{n}\right){% endequation %} פתירה אז {% equation %}a\in S{% endequation %}, ממש על פי הגדרה.

האתגר הוא הכיוון השני. נניח ש-{% equation %}S_{q}{% endequation %} מוגדרת על ידי {% equation %}q{% endequation %} בצורה ה"ישנה"; איך נמצא פולינום {% equation %}p{% endequation %} שמגדיר אותה בצורה ה"חדשה"? ובכן, נניח ש-{% equation %}a\in S{% endequation %}. אז {% equation %}q\left(a,y_{1},\dots,y_{m}\right){% endequation %} פתיר - אפשר לקבל ממנו אפס. לכן {% equation %}p\left(y_{1},\dots,y_{m}\right)=a-q\left(a,y_{1},\dots,y_{m}\right){% endequation %} הולך להחזיר {% equation %}a{% endequation %} כאשר מציבים בו בדיוק את הערכים של ה-{% equation %}y{% endequation %}-ים שמאפסים את {% equation %}q\left(a,y_{1},\dots,y_{m}\right){% endequation %}. אבל זה לא מספיק טוב, כי עבור ערכים שונים של ה-{% equation %}y{% endequation %}-ים {% equation %}q{% endequation %} לא בהכרח יתאפס ואז {% equation %}a-q\left(a,y_{1},\dots,y_{m}\right){% endequation %} יהיה מספר לא קשור. רק מה, צריך לזכור שמותר ל-{% equation %}p{% endequation %} להחזיר מספרים לא קשורים, בתנאי שהם <strong>שליליים</strong> (או אפס) אז בואו נוודא שנחזיר מספר שלילי, פשוט על ידי כוח גס: {% equation %}p\left(y_{1},\dots,y_{m}\right)=a\left(1-q^{2}\left(a,y_{1},\dots,y_{m}\right)\right){% endequation %}. הבהירו לעצמכם למה הפולינום הזה עושה את העבודה.

אבל רגע, עוד לא סיימנו! הפולינום שלנו עושה את העבודה עבור ערך ספציפי של {% equation %}a{% endequation %}, ואנחנו רוצים שהוא יעבוד לכל {% equation %}a{% endequation %}. זו לא בעיה, כמובן: הפולינום ה"נכון" הוא {% equation %}p\left(x,y_{1},\dots,y_{m}\right)=x\left(1-q^{2}\left(x,y_{1},\dots,y_{m}\right)\right){% endequation %}, ועכשיו באמת סיימנו את הוכחת המשפט.

בואו נקפוץ לרגע לתוצאה שנובעת מהמשפט הזה, ומהטענה הכללית בהרבה (שהיא היעד הסופי שלנו בסדרת הפוסטים הזו) לפיה כל קבוצה הניתנת למניה רקורסיבית היא דיופנטית; בפרט הדבר נכון לקבוצת הראשוניים (כי יש אלגוריתם שבודק אם מספר הוא ראשוני או לא). לכן קיים פולינום {% equation %}p{% endequation %}, כך שכאשר מציבים בו מספרים שלמים, הערכים החיוביים שמתקבלים הם <strong>בדיוק</strong> כל המספרים הראשוניים! התוצאה הזו היא הדבר הכי קרוב שאני מכיר ל"נוסחה עבור הראשוניים". יש אפילו בניות קונסטרוקטיביות של פולינום {% equation %}p{% endequation %} כזה (שנראות די מפלצתיות ולא אציג כאן). עדיין, קשה להגיד שזו תוצאה שימושית במיוחד בגלל האופן האקראי משהו שבו מקבלים ראשוניים מתוך הנוסחה הזו - מציבים ערכים ב-{% equation %}p{% endequation %} ומתפללים שיצא משהו חיובי. אף אחד לא ערב לכך שזה יקרה רוב הזמן, ושהראשוני שנמצא יהיה מעניין (כלומר, מתאים לאי-אלו קריטריונים שיש לנו). כמובן, "ראשוניים" כאן זה סתם בגלל שהראשוניים הם אחת מהקבוצות הפופולריות ביותר של מספרים טבעיים, אם לא הפופולרית מכולן.

הבנו קבוצות דיופנטיות? יפה. אז בואו נעבור לסוג מיוחד של קבוצה - פונקציה. פונקציה היא יחס - קבוצה של איברים מהצורה {% equation %}\left(x_{1},\dots,x_{n},y\right){% endequation %} - עם התכונה המיוחדת שלכל {% equation %}x_{1},\dots,x_{n}{% endequation %} <strong>קיים</strong> {% equation %}y{% endequation %} כך ש-{% equation %}\left(x_{1},\dots,x_{n},y\right){% endequation %} שייך לקבוצה, ואותו {% equation %}y{% endequation %} הוא יחיד. חשבו על {% equation %}\left(x_{1},\dots,x_{n}\right){% endequation %} בתור "קלט" ועל {% equation %}y{% endequation %} בתור "פלט" של הפונקציה. לרוב מסמנים פונקציה ב-{% equation %}f{% endequation %}, ואז במקום לומר ש-{% equation %}\left(x_{1},\dots,x_{n},y\right){% endequation %} שייך לפונקציה, כותבים {% equation %}f\left(x_{1},\dots,x_{n}\right)=y{% endequation %}. רובכם בוודאי יודעים את זה (אבל זכרו שלא כל מי שמכיר פונקציות גם מכיר את הגדרתן הפורמלית, התורת-קבוצתית).

ובכן, פונקציה היא דיופנטית אם כשהיא שמה עליה את הכובע של "קבוצה", היא קבוצה דיופנטית. <strong>שימו לב מה זה לא אומר</strong>! זה לא אומר שפונקציה דיופנטית היא בהכרח פולינום! לא, לא לא! למעשה, הדבר החשוב ביותר שנעשה בכל סדרת הפוסטים הזו יהיה להוכיח שהפונקציה האקספוננציאלית היא דיופנטית.

עכשיו אפשר לעצור לרגע ולראות את היעד הסופי שלנו. המטרה שלנו היא להוכיח שקבוצות דיופנטיות וקבוצות הניתנות למניה רקורסיבית הן אותו דבר. כדי לעשות את זה, אני צריך לתת הגדרה פורמלית ל"קבוצות הניתנות למניה רקורסיבית". ההגדרה הסטנדרטית היא באמצעות מכונות טיורינג, אבל לצורך שלנו הרבה יותר נוח יהיה להשתמש במודל חישובי אחר, כזה שהוא הרבה יותר דומה באופיו לפונקציות הדיופנטיות מלכתחילה - <strong>פונקציות רקורסיביות</strong>. התוצאה שאני חותר אליה היא שפונקציה היא רקורסיבית אם ורק אם היא דיופנטית; ראשית כל עלינו להבין, אם כן, מהי פונקציה רקורסיבית. נתחיל עם מה שלא: זו <strong>לא</strong> פונקציה ש"קוראת לעצמה"! המילה "רקורסיה" כאן קיימת מסיבות היסטוריות (ובקרוב נבין את המוטיבציה לה), אבל שם הרבה יותר מתאים לפונקציות רקורסיביות הוא פשוט "פונקציות ניתנות לחישוב".

מאיפה צצו הפונקציות הרקורסיביות? שאלה טובה. הן מופיעות במאמר המפורסם של קורט גדל שבו הוא הוכיח את משפטי אי השלמות שלו, וכבר הזכרתי אותן <a href="http://www.gadial.net/2009/05/10/godel_incompleteness_proof_sketch/">בפוסט שעסק בהוכחה</a> ולמיטב הבנתי גדל הוא זה שהמציא את הניסוח הזה שלהן, אבל אולי אני טועה. עם זאת, מה שהופיע במאמר של גדל לא היה שלם - היה חסר כלל אחד שהופיע רק מאוחר יותר, ואפרט על כך עוד מעט.

הפונקציות הרקורסיביות מוגדרות על ידי אוסף של פונקציות <strong>בסיס</strong> פשוטות ביותר, ש"ברור" לנו שאפשר לחשב אותן, ועוד כמה <strong>פעולות יצירה</strong> שמאפשרות ליצור פונקציות חדשות על בסיס פונקציות קיימות, וגם אותן "ברור" לנו שניתן לחשב. מה פשר ה"ברור" הזה? בימינו, זה אומר שאפשר לכתוב תוכנית מחשב שמבצעת את החישובים הללו. זה מראה שמחשבים מודרניים חזקים <strong>לפחות</strong> כמו פונקציות רקורסיביות - כל פונקציה רקורסיבית היא ניתנת לחישוב בידי מחשב מודרני. מצד שני, וזה הרבה יותר מפתיע, גם כל פונקציה שמחשב מודרני יכול לחשב היא רקורסיבית. כדי להוכיח את זה צריך לעבוד לא מעט ואני לא הולך לעשות זאת כאן (אבל יום אחד אני מקווה לעשות את זה).

כל הפונקציות הרקורסיביות הן פונקציות מהטבעיים החיוביים לטבעיים החיוביים; לכאורה זה קצת מגביל אותנו כי הרי מחשב יכול להתעסק גם במספרים שליליים ובדברים שהם בכלל לא מספרים, אבל מכיוון שהכל ניתן לקידוד בתור מספר טבעי חיובי (כל מידע שמיוצג במחשב, לא <strong>כל </strong>דבר בעולם!) זה לא באמת מפריע לנו.

	<li>הפונקציה הקבועה {% equation %}c\left(x\right)=1{% endequation %}.</li>
	<li>פונקציית העוקב {% equation %}s\left(x\right)=x+1{% endequation %}</li>
	<li>פונקצית ההטלה, {% equation %}U_{i}^{n}\left(x_{1},\dots,x_{n}\right)=x_{i}{% endequation %} (המוגדרת לכל {% equation %}n,i{% endequation %}).</li>
</ol>
טוב, עבור פונקציות הבסיס הללו לא קשה לראות שהן דיופנטיות. הפונקציה הראשונה מוגדרת על ידי:

{% equation %}y=c\left(x\right)\iff\left(y=1\right){% endequation %}

השניה מוגדרת על ידי:

{% equation %}y=s\left(x\right)\iff\left(y=x+1\right){% endequation %}

והשלישית מוגדרת על ידי:

{% equation %}y=U_{i}^{n}\left(x_{1},\dots,x_{n}\right)\iff\left(y=x_{i}\right){% endequation %}

טוב, זה היה מטופש למדי. איפה האתגר? בהמשך.

הפעולה הראשונה שממנה בונים פונקציות רקורסיביות חדשות מתוך הקיימות היא <strong>הרכבה</strong>. היא מוגדרת כך: אם {% equation %}g_{1},\dots,g_{m}{% endequation %} הן פונקציות רקורסיביות על המשתנים {% equation %}x_{1},\dots,x_{n}{% endequation %} ואילו {% equation %}f{% endequation %} היא פונקציה רקורסיבית על {% equation %}m{% endequation %} משתנים, אז הפונקציה {% equation %}h\left(x_{1},\dots,x_{n}\right)=f\left(g_{1}\left(x_{1},\dots,x_{n}\right),\dots,g_{m}\left(x_{1},\dots,x_{n}\right)\right){% endequation %} גם היא רקורסיבית. מה שאנחנו רוצים להראות הוא שאותו הדבר קורה עם פונקציות דיופנטיות, כלומר שאם {% equation %}g_{1},\dots,g_{m},f{% endequation %} כולן דיופנטיות, כך גם {% equation %}h{% endequation %}.

למרבה המזל, את זה קל להראות כבר עם מעט התכונות שראינו של פונקציות דיופנטיות:

{% equation %}y=h\left(x_{1},\dots,x_{n}\right)\iff\exists t_{1},\dots,t_{m}\left(y=f\left(t_{1},\dots,t_{m}\right)\wedge t_{1}=g_{1}\left(x_{1},\dots,x_{n}\right)\wedge\dots\wedge t_{m}=g_{m}\left(x_{1},\dots,x_{n}\right)\right){% endequation %}

מה בעצם עשינו כאן? אגף ימין מגדיר פולינום גדול, שמקבל בתור משתנים את {% equation %}x_{1},\dots,x_{n}{% endequation %} וגם את המשתנים {% equation %}t_{1},\dots,t_{m}{% endequation %}. עכשיו הוא לוקח את הפולינום שמגדיר את {% equation %}f{% endequation %} (כשהמשתנים בו מתאימים ל-{% equation %}t_{1},\dots,t_{m}{% endequation %} ו-{% equation %}y{% endequation %}), את הפולינום שמגדיר את {% equation %}g_{1}{% endequation %} (כשהמשתנים בו מתאימים ל-{% equation %}x_{1},\dots,x_{n}{% endequation %} ו-{% equation %}t_{1}{% endequation %}), את הפולינום שמגדיר את {% equation %}g_{2}{% endequation %} (כבר הבנתם...?) וכן הלאה עד {% equation %}g_{m}{% endequation %}, ואז מחבר את הריבועים של כולם (זו מהות ה-{% equation %}\wedge{% endequation %} שעושים על כולם).

כן, זה כבר לא מובן מאליו, וזה יסתבך בהמשך. לכן כדאי לנקוט גישה קצת יותר כללית לעניין הזה. אפשר לגייס כאן את הלוגיקה המתמטית לעזרתנו; הרי מה שעשינו לעיל הוא פשוט לכתוב פסוק מסויים ב<a href="http://www.gadial.net/2012/06/17/first_order_logic/">לוגיקה מסדר ראשון</a> שאיכשהו מובטח לנו שהוא מגדיר פונקציה דיופנטית. הפסוק הזה הכיל סימנים לפונקציות דיופנטיות, את סימן השוויון, את סימן ה"וגם" ואת סימן ה"קיים". האם אפשר להשתמש בעוד דברים? כן, וחיזוק ה"שפה" שלנו הוא היעד הטכני המרכזי שלנו. ארחיב על כל זה בצורה מסודרת כשהזמן יגיע.

לבינתיים, הבה ונראה את הפעולה השניה שמשמשת לבנית פונקציות רקורסיביות: <strong>רקורסיה פרימיטיבית</strong>. מכאן הגיע השם "רקורסיבי" לפונקציות הללו (כי אצל גדל זו הייתה פעולת הבניה החשובה באמת, שנתנה למחלקת הפונקציות הזו את המורכבות שלה). זו לא פעולה קלה לעיכול מבחינה פורמלית, אבל הרעיון הבסיסי פשוט: אנחנו מגדירים פונקציה באמצעות ערכים קטנים יותר שלה. זה הכל.

רקורסיה פרימיטיבית מוגדרת כך: בהינתן פונקציה רקורסיבית {% equation %}f{% endequation %} על {% equation %}n{% endequation %} משתנים ("תנאי ההתחלה"), ופונקציה רקורסיבית {% equation %}g{% endequation %} על {% equation %}n+2{% endequation %} משתנים ("צעד הרקורסיה"), אנחנו מגדירים פונקציה חדשה, {% equation %}h{% endequation %}, על {% equation %}n+1{% endequation %} משתנים (שעליהם כדאי לחשוב כך: {% equation %}n{% endequation %} המשתנים הראשונים הם "פרמטרים" קבועים ואילו המשתנה האחרון הוא מה שעליו מוגדרת הרקורסיה), באופן הבא:

{% equation %}h\left(x_{1},\dots,x_{n},1\right)=f\left(x_{1},\dots,x_{n}\right){% endequation %}

{% equation %}h\left(x_{1},\dots,x_{n},t+1\right)=g\left(t,h\left(x_{1},\dots,x_{n},t\right),x_{1},\dots,x_{n}\right){% endequation %}

מה קורה כאן? הערך של {% equation %}h{% endequation %} ב"התחלה", כשהמשתנה האחרון הוא 1, נקבע על ידי הפרמטרים ופונקצית תנאי ההתחלה. בהמשך, הערך של {% equation %}h{% endequation %} עבור ערך כלשהו של המשתנה האחרון נקבע על ידי {% equation %}g{% endequation %}, כשהיא מביאה בחשבון את הערך הקודם של המשתנה האחרון, הערך ש-{% equation %}h{% endequation %} החזירה על הערך הקודם הזה, והפרמטרים. זה נראה קצת מפחיד אבל זה לא באמת נורא.

איך מראים שהפעולה הזו משמרת דיופנטיות? או, זה כבר קצת מסובך. למרבה המזל, אפשר לשאוב השראה מקורט גדל עצמו; במאמר שלו הוא הוכיח שאפשר להגדיר את הפעולה הזו בשפה מסדר ראשון מסויימת; למרות שהשפה שלנו היא קצת שונה, אפשר יהיה לשאוב השראה ורעיונות מהטכניקות שלו. את זה נעשה בפוסט המשך; לעת עתה בואו נחשוב על נסיון נאיבי ואיך הוא נכשל:

{% equation %}y=h\left(x_{1},\dots,x_{n},z\right)\iff\exists t_{1},t_{2},\dots,t_{z}(t_{1}=f\left(x_{1},\dots,x_{n}\right)\wedge{% endequation %}

{% equation %}\wedge t_{2}=g\left(1,t_{1},x_{1},\dots,x_{n}\right)\wedge t_{3}=g\left(2,t_{2},x_{1},\dots,x_{n}\right)\wedge\dots{% endequation %}

{% equation %}\wedge t_{z}=g\left(z-1,t_{z-1},x_{1},\dots,x_{n}\right)\wedge y=t_{z}){% endequation %}

מה הולך כאן? אנחנו אומרים ש-{% equation %}y=h\left(x_{1},\dots,x_{n},z\right){% endequation %} רק אם קיימת <strong>סדרה</strong> {% equation %}t_{1},\dots,t_{z}{% endequation %} שהאיבר הראשון בה הוא {% equation %}f\left(x_{1},\dots,x_{n}\right){% endequation %} - הערך ההתחלתי הנכון, בהינתן הפרמטרים - וכל איבר בה נובע מקודמו על ידי {% equation %}g{% endequation %}, וכמו כן האיבר האחרון הוא {% equation %}y{% endequation %}. מבחינה רעיונית הכל נכון; הבעיה היא סינטקטית - הפסוק שכתבתי כאן לא מתאים ל"שפה" שהצגתי עד כה, מה שאומר שעל פניו לא ברור אם אפשר איכשהו לתרגם אותו חזרה לפולינום שמגדיר פונקציה דיופנטית. בעיה מהותית אחת היא שאורך הפסוק <strong>אינו קבוע</strong>! הוא תלוי ב-{% equation %}z{% endequation %}, כי ככל ש-{% equation %}z{% endequation %} יותר גדול, יש יותר משתנים {% equation %}t_{1},\dots,t_{z}{% endequation %}. אם עבור ערכים שונים של {% equation %}z{% endequation %} יש פסוקים שונים, זה אומר שגם יהיו פולינומים שונים, אבל אנחנו צריכים פולינום יחיד שמטפל <strong>בכל</strong> {% equation %}z{% endequation %}. לכן הבניה הזו לא מוצלחת ונזדקק לפיתוח של כלי טכני חדש - פונקציה דיופנטית שמסוגלת לקודד סדרות שרירותיות - על מנת להציע בניה טובה יותר, וגם זה לא יספיק לנו. אבל על כך נדבר בהמשך.

בואו נסיים את הצגת הפונקציות הרקורסיביות עם פעולת היצירה האחרונה. הפעולה הזו לא הופיעה במאמר של גדל; גדל לא נזקק לה לצורך משפטי אי השלמות. הצורך בה צץ רק כאשר החלו לחשוב על הפונקציות הרקורסיביות כמתארות את "כל מה שניתן לחשב", שכן קיימות פונקציות שבבירור ניתן לחשב אותן (כלומר, קיים אלגוריתם שעושה זאת) אבל הן אינן ניתנות לייצור באמצעות כללי היצירה שהראיתי עד כה. בעתיד אני מקווה להראות את הדוגמה הקלאסית ביותר - <strong>פונקציית אקרמן</strong>, שגדלה "מהר מדי" מכדי שתהיה ניתנת לתיאור באמצעות כללי היצירה הנוכחיים.

אם כן, פונקציות שניתנות ליצירה באמצעות כללי היצירה הנוכחיים מתוך פונקציות הבסיס נקראות <strong>פונקציות פרימיטיביות רקורסיביות</strong>, וקיימות פונקציות ניתנות לחישוב שאינן פרימיטיביות רקורסיביות. הכלל שפותר את הבעיה הזו הוא כלל שמאפשר לבצע "חיפוש בלתי חסום": אם נתונות פונקציות רקורסיביות {% equation %}f,g{% endequation %} על {% equation %}n+1{% endequation %} משתנים, אז מוגדרת באמצעותן פונקציה רקורסיבית {% equation %}h{% endequation %} על {% equation %}n{% endequation %} משתנים באופן הבא:

{% equation %}h\left(x_{1},\dots,x_{n}\right)=\min_{y}\left(f\left(x_{1},\dots,x_{n},y\right)=g\left(x_{1},\dots,x_{n},y\right)\right){% endequation %}

כמקודם, חשבו על {% equation %}x_{1},\dots,x_{n}{% endequation %} בתור "פרמטרים". אחרי שקבענו אותם, אנחנו יכולים לחפש את ה-{% equation %}y{% endequation %} המינימלי שעבורו {% equation %}f{% endequation %} תהיה שווה ל-{% equation %}g{% endequation %}, וזה הערך ש-{% equation %}h{% endequation %} תחזיר על אותם פרמטרים. אלא שכאן מייד צצה השאלה הפשוטה - מה קורה אם בכלל לא קיים {% equation %}y{% endequation %} כזה? התשובה היא שבמקרה זה {% equation %}h{% endequation %} תהיה <strong>לא מוגדרת</strong> על הקלט. מרגע שהכנסו את הכלל החדש הזה, שנקרא <strong>כלל המינימיזציה</strong> לתמונה, גרמנו לכך שקבוצת הפונקציות הרקורסיביות תכלול גם פונקציות <strong>חלקיות</strong>, כאלו שלא מוגדרות לכל הקלטים (ולמעשה, אפילו הפונקציה שאינה מוגדרת לאף קלט היא רקורסיבית; פשוט קחו את {% equation %}f{% endequation %} להיות הפונקציה שמחזירה תמיד 1 ואת {% equation %}g{% endequation %} להיות הפונקציה שמחזירה תמיד 2). מה שמעניין כאן הוא שפונקציית אקרמן שהזכרתי לעיל כן מוגדרת לכל קלט, כלומר כלל המינימיזציה הכרחי כדי שנוכל "לתפוס" את כל הפונקציות הניתנות לחישוב שמוגדרות לכל קלט; זה לא שהוספנו אותו רק כדי להיות מסוגלים לטפל בפונקציות חלקיות.

טוב, אז הבנו בערך מה זה הכלל הזה; איך אפשר להראות שאם {% equation %}f,g{% endequation %} דיופנטיות, כך גם {% equation %}h{% endequation %}?

שוב, מבחינה רעיונית הפסוק שמגדיר את התכונה הזו אינו מסובך:

{% equation %}y=h\left(x_{1},\dots,x_{n}\right)\iff f\left(x_{1},\dots,x_{n},y\right)=g\left(x_{1},\dots,x_{n},y\right)\wedge{% endequation %}

{% equation %}\wedge\forall t\left(t&lt;y\to f\left(x_{1},\dots,x_{n},y\right)\ne g\left(x_{1},\dots,x_{n},y\right)\right){% endequation %}

אבל שימו לב לשימוש ב-{% equation %}\forall{% endequation %} ("לכל") כאן. אני כבר יכול לומר ש<strong>אין שום סיכוי</strong> שבשפה שלנו נוכל להשתמש ב-{% equation %}\forall{% endequation %} בצורה חופשית ועדיין לקוות שאפשר יהיה לתרגם את הסיפור חזרה לפונקציה דיופנטית. מה שכן נוכל לעשות - וזה לב ליבה של המטרה הטכנית שלנו - הוא להשתמש ב-{% equation %}\forall{% endequation %} <strong>חסום</strong>, כלומר במקום לומר "לכל {% equation %}t{% endequation %}", לומר "לכל {% equation %}t{% endequation %} שקטן מ-{% equation %}y{% endequation %}", שזה בדיוק מה שאנחנו צריכים כאן, בעצם.

אם כן, עכשיו היעדים הטכניים שלנו קצת יותר ברורים. יש לנו שתי פעולות יצירה שאנחנו רוצים להוכיח שמשמרות דיופנטיות של פונקציות. האופן הטכני שבו נעשה זאת הוא להגדיר שפה מסדר ראשון מסויימת, שבה מותר להשתמש בקבוצה מאוד מוגדרת ומדויקת של סימנים, כך שכל פונקציה שמוגדרת באמצעות השפה הזו היא אוטומטית דיופנטית. כדי שהשפה הזו תהיה חזקה אנחנו מוסיפים לה שני כלי עבודה מרכזיים - האחד הוא פונקציה לקידוד סדרות סופיות, והשני הוא כמת "לכל" חסום. בשביל לטפל בפונקציה לקידוד סדרות סופיות יש להשתמש במשפט מתמטי נפוץ שחביב עלי מאוד - <strong>משפט השאריות הסיני</strong>, ולכן אקדיש לו את הפוסט הבא.
